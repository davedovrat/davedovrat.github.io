Git - GitLab - Github - Hurray!

I like GitLab because

  • they have a nice scrum board and labeling system.
  • you can tag issues in your commit messages and they automatically update the scrum board issue card.
  • nice markdown support for in-project documentation.
  • free CI/CD and static web-hosting!

Any public repo should have

  • a .gitignore file.
  • a LICENSE.
  • a nice README.MD.

The Takeaway

The Takeaway

  • Git is Great!
  • The vanilla Git-Gui is all you really need to collaborate or work alone...
  • ...but Git[Hub, Lab] are good platforms for managing your project and sharing your code.
  • We will be using GitHub.
